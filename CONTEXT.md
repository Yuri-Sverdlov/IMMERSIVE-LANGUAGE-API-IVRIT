## ~~План TASK-008~~ — **сделано (2026-06-23)**

> Запрос пользователя 2026-06-22: **скорость ответа ИИ в диапазоне 0,7 … 1,5**
> (1,0 = как сейчас; меньше — медленнее, больше — быстрее).
> Реализовано: `speech_playback_rate` в `immergo.config.json`, clamp на backend,
> resample в `playback.worklet.js`. Архив: `tasks/done/008-speech-playback-rate/`.

### Что пользователь имеет в виду

Речь о **темпе озвучки ответа модели**, а не о том, как быстро ИИ «начинает говорить» после паузы.
Последнее уже настраивается через `silence_duration_ms` (VAD) в `immergo.config.json`.

### Что проверили в коде и API

| Механизм | Где | Вывод |
|----------|-----|-------|
| Gemini Live `SpeechConfig` | `geminilive.js` → setup | Есть только `voice_name` (сейчас `"Puck"`) и язык. **Поля `speaking_rate` / скорости речи в Live API нет.** |
| `silence_duration_ms` | config → `automaticActivityDetection` | Это **пауза до ответа** (VAD), не скорость речи. Не путать с TASK-008. |
| Воспроизведение PCM | `mediaUtils.js` → `AudioPlayer`, worklet `playback.worklet.js` | Аудио 24 kHz PCM16 → Float32, **без ускорения/замедления**. Есть только `setVolume()`. |

### Решение архитектора (зафиксировано для TASK-008)

1. **Не через Gemini API** — параметра скорости речи в setup нет; менять промпт («говори медленнее») ненадёжно.
2. **На клиенте при воспроизведении** — добавить в `immergo.config.json`:
   ```json
   "speech_playback_rate": 1.0
   ```
   Допустимый диапазон: **0,7 … 1,5** (clamp в коде).
3. **Реализация (черновик):** читать rate из `/api/config`; в `AudioPlayer` или playback worklet
   воспроизводить буфер быстрее/медленнее (resample / изменение effective sample rate / очередь с другим темпом).
   Точный способ — на усмотрение кодера; главное — без артефактов и без изменения pitch (желательно).
4. **Backend:** прокинуть поле в `GET /api/config` и дефолт в `load_immergo_config()`.
5. **Документация:** строка в `QUICK_START.md` (таблица «Ограничения»).

## План TASK-013 — «Микрофон: вкл/выкл» (идея пользователя, 2026-06-24)

> **Статус:** только план, **код не писать** до отдельного `tasks/TASK.md`.
> Связано с захватом микрофона (`AudioStreamer`, `mediaUtils.js`) и VAD (`silence_duration_ms`, TASK-007).

### Запрос пользователя

- **Суть:** во время разговора с LLM-партнёром иногда нужно, чтобы модель **не слышала** звуки в комнате.
- **Элемент UI:**
  - кнопка/переключатель с текстом **«Микрофон: включен»** / **«Микрофон: отключен»** (toggle);
  - рядом **кружок** небольшого диаметра — индикатор:
    - **красный** — микрофон **включён** (звук уходит к модели);
    - **серый** — микрофон **отключён**.
- **Поведение «отключен»:**
  1. PCM с микрофона **не отправляется** в Gemini (LLM не слышит комнату).
  2. **Отключается контроль тишины (VAD)** — сейчас `silence_duration_ms: 2000` в `immergo.config.json` / `automatic_activity_detection`.
     Модель не должна «реагировать на паузу 2 сек» пока пользователь в режиме mute.
- **Поведение «включен» (возврат):**
  - снова отправлять аудио;
  - **восстановить** `silence_duration_ms` из config (2000 ms или значение из `/api/config`).

**Сохранение между сессиями:** пользователь **не просил** — по умолчанию при новой миссии микрофон **включён**; уточнить при TASK при необходимости.

### Решение архитектора (черновик UI)

**Размещение:** **правая часть** той же панели — см. **«Единая панель управления сессией»** выше. ~~Над user visualizer / у mic-CTA~~ — **не** делать.

```
( ● )  Микрофон: включен
```

- `●` — круг ~12–16 px: `#f44336` (red) = on, `#9e9e9e` (gray) = off.
- Клик по строке или отдельной кнопке — toggle.
- Виден **после старта voice-сессии** (когда `AudioStreamer` активен); до старта — опционально disabled/hidden (на усмотрение TASK).

**Технический черновик (для кодера):**

| Задача | Где | Идея |
|--------|-----|------|
| Не слать звук | `AudioStreamer` (`mediaUtils.js`) | Флаг `muted` / `isStreaming=false` без разрыва WebSocket — не вызывать `sendAudioMessage` |
| VAD off | `GeminiLiveAPI.automaticActivityDetection` (`geminilive.js`) | При mute: `disabled: true` **или** эквивалент; при unmute: `disabled: false` + `silence_duration_ms` из сохранённого config |
| Восстановление 2 s | `view-chat.js` | Хранить `_savedSilenceDurationMs` при первом connect; при unmute → `setSilenceDurationMs(saved)` |
| Mid-session update | `geminilive.js` | Setup уходит один раз при connect — **проверить**, можно ли обновить `realtime_input_config` mid-session; если нет — mute достаточно на **клиенте** (не слать PCM) + исследовать server-side VAD off |

**Файлы (ожидаемо при реализации):**
- `app/src/components/view-chat.js` — UI + toggle logic;
- `app/src/lib/gemini-live/mediaUtils.js` — `setMuted()` / pause send;
